# OMNI tool-scoring SOUL

OMNI is a manifest-conditioned behavior-cloning policy. The observation trunk maps the normalized schema-v2 observation (`obs`, `[B,129]`) to a context vector. Each currently afforded tool is represented by one row in `tool_descriptors` (`[B,T,D]`); a descriptor trunk maps rows to vectors, and a pairwise score head scores `(context, descriptor)` independently. `afforded` (`[B,T]`) hard-masks unavailable rows. `T` is dynamic in the exported ONNX graph: adding a tool means appending a descriptor row and mask bit, not changing a fixed output classifier or retraining its output layer.

The target head maps the shared context to the eight observed-neighbor slots (`target_logits`, `[B,8]`). The parameter head emits a small continuous parameter vector (`params`, `[B,4]`) for the selected tool. ONNX inputs are `obs`, `tool_descriptors`, and `afforded`; outputs are `tool_scores`, `target_logits`, and `params`. Batch and tool axes are dynamic, and all descriptor construction is deterministic (stable BLAKE2s features, never Python's randomized `hash`).

`mw_training.train_omni` trains on schema-v2 trajectory records or `training/artifacts/fixtures.json`, prints per-epoch loss/match history, and exports the graph. `OmniDataset.from_fixtures` derives teacher labels from the existing teacher logits solely for the CPU smoke fixture; trajectory training reads the teacher `decision` fields.
The existing Rust `mw-neural::NeuralRuntime` intentionally remains the fixed-vocabulary compatibility path (`obs` + `afforded_mask`). Until its dynamic-manifest API is adopted, OMNI artifacts use the documented `onnxruntime` adapter (`onnxruntime.InferenceSession`, `CPUExecutionProvider`) with the three inputs above; this adapter was smoke-tested with `B=2,T=5` and produced `(2,5)`, `(2,8)`, `(2,4)` outputs. No model output is treated as kernel truth: callers still submit validated intents.
